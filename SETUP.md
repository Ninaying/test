# 🌲 木材庫存系統 — 安裝設定指南

## 事前準備
- 一個 Google 帳號
- 一份 Google 試算表（空白即可）
- 一個用於存放照片的 Google 雲端硬碟資料夾

---

## 步驟一 — 取得 ID

### Google 試算表 ID
1. 開啟（或建立）一份 Google 試算表。
2. 從網址列複製 `/d/` 與 `/edit` 之間的那一段字串：
   ```
   https://docs.google.com/spreadsheets/d/1ABC123XYZ.../edit
                                          ^^^^^^^^^^^
                                          這就是您的 SHEET_ID
   ```

### Google 雲端硬碟資料夾 ID
1. 開啟（或建立）一個雲端硬碟資料夾。
2. 複製網址最末段的字串：
   ```
   https://drive.google.com/drive/folders/1ABC123XYZ...
                                           ^^^^^^^^^^^
                                           這就是您的 FOLDER_ID
   ```

---

## 步驟二 — 建立 Google Apps Script 專案

1. 前往 [script.google.com](https://script.google.com)，點選 **新增專案**。
2. 為專案命名（例：`木材庫存 API`）。
3. 刪除預設的 `myFunction` 程式碼。
4. 將 **`Code.gs` 的完整內容** 貼入編輯器。
5. 將檔案頂部的兩個常數替換為您自己的 ID：
   ```js
   const SHEET_ID  = "貼上您的試算表 ID";
   const FOLDER_ID = "貼上您的資料夾 ID";
   ```
6. 按下 **儲存**（Ctrl+S / Cmd+S）。

---

## 步驟三 — 部署為網頁應用程式

1. 點選 **部署 → 新增部署作業**。
2. 點選「選取類型」旁的 ⚙️ 齒輪圖示 → 選擇 **網頁應用程式**。
3. 填寫以下設定：

   | 設定項目 | 值 |
   |---|---|
   | 說明 | `木材庫存 v1` |
   | 執行身分 | **我（您的 Google 帳號）** |
   | 存取對象 | **所有人**（請見下方安全性說明） |

4. 點選 **部署**。
5. 點選 **授權存取**，並授予所需權限。
6. **複製網頁應用程式網址**，格式如下：
   ```
   https://script.google.com/macros/s/AKfycb.../exec
   ```
   請將此網址貼入應用程式的網址欄位。

> **安全性說明：** 設定為「所有人」表示任何知道此網址的人皆可傳送資料。
> 正式上線環境建議在 `doPost` 中加入共用金鑰驗證機制。

---

## 步驟四 — 開啟前端頁面

在平板的任何現代瀏覽器中開啟 `index.html`。

> **小提示：** 若需完整離線使用，可將頁面儲存至本機，或透過簡易本機伺服器提供服務（`python3 -m http.server`）。應用程式本身不需要 Node.js 或任何建置步驟。

---

## 步驟五 — 設定應用程式

1. 在頁面頂部，將您的 **網頁應用程式網址** 貼入網址欄位。
2. 系統會自動將網址儲存至 `localStorage`，每台裝置只需設定一次。
3. 成功連線後，狀態標示會轉為 **綠色**。

---

## 步驟六 — 修改程式碼後重新部署

若您在初次部署後修改了 `Code.gs`，必須建立 **新的部署作業** 才能讓變更生效：

1. 點選 **部署 → 新增部署作業**（請勿點選「管理部署作業」）。
2. 複製新的網址並更新至應用程式中。

> 每次部署都會產生新網址。「管理部署作業 → 編輯」只會更新現有版本，使用最新版本網址才是最安全的做法。

---

## 離線同步運作方式

| 情境 | 行為 |
|---|---|
| 儲存時網路正常 | 記錄直接上傳至 Google 試算表與雲端硬碟 |
| 儲存時無網路 | 記錄儲存至瀏覽器 `localStorage` 佇列 |
| 恢復連線後點選「同步」 | 所有佇列記錄透過 `syncBatch` 批次上傳 |
| 部分同步失敗 | 失敗的記錄留在佇列，成功的標記為已同步 |

照片以 base64 字串形式儲存在離線佇列中，同步時才上傳至雲端硬碟。**請避免在佇列中堆積過大的圖片**（每張超過 3 MB）—— `localStorage` 上限約為 5 MB。高用量場景建議縮小相機拍攝尺寸或壓縮後再上傳。

---

## Google 試算表欄位結構

腳本會自動建立名為「**庫存**」的工作表，包含以下欄位：

| A | B | C | D | E | F |
|---|---|---|---|---|---|
| 時間戳記 | 重量（公斤） | 木材樹種 | 備註 | 自訂欄位 | 圖片網址 |

自訂欄位以 `名稱：值 | 名稱：值` 的豎線分隔格式儲存於 E 欄。

## 疑難排解

| 問題 | 解決方式 |
|---|---|
| 有網路但狀態顯示「離線」 | 確認 GAS 網址正確，且部署設定為「所有人」可存取 |
| 同步時出現 `403 禁止存取` | 重新部署 GAS 並將存取設為「所有人」，更新網址 |
| 照片未出現在雲端硬碟 | 確認 `FOLDER_ID` 正確，且 GAS 帳號有該資料夾的寫入權限 |
| `localStorage` 額度不足錯誤 | 在應用程式中清除舊記錄，或在同步前縮小圖片數量/尺寸 |
| GAS 修改後未生效 | 必須建立「新的部署作業」（不可只編輯現有部署） |
